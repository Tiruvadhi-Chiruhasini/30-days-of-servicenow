# Day 1 of 30 Days of ServiceNow 🚀

**Date:** 29/05/2026
**Instance:** `devXXXXX.service-now.com`

## Claimed my first ServiceNow instance

Today was mostly about exploring the platform instead of blindly watching tutorials.

* Claimed a free **Personal Developer Instance (PDI)** from developer.servicenow.com
* Logged in as admin
* No setup drama, no installations, everything runs directly in the browser
* ServiceNow already comes with sample users, incidents, and demo data which honestly makes learning easier

## First impressions while exploring 👀

The platform felt huge at first but after clicking around, the structure started making sense.

### Things I understood:

* The **All menu** is basically the control center for navigating applications/modules
* Almost everything in ServiceNow is:

  * a **List** → collection of records
  * or a **Form** → single record view
* Global search searches across the whole platform, not just one section
* Incident records can be accessed from:
  `Incident → All`

## Created my first Incident 🎯

Created my first incident ticket for a VPN issue.

### Incident details:

* Short description:
  `Unable to connect to VPN from home network`
* Added caller information
* Set urgency
* Submitted the incident successfully

### Interesting thing I noticed:

When Impact/Urgency values change, **Priority auto-calculates automatically**.

That’s my first real exposure to ServiceNow automation logic instead of just theory.

## Concepts I learned naturally while doing

| Term          | Meaning                                          |
| ------------- | ------------------------------------------------ |
| PDI           | Personal Developer Instance (free sandbox)       |
| Application   | A functional area like Incident Management       |
| Module        | Menu items inside applications                   |
| Record        | One entry in a table                             |
| Field         | Individual data values like Priority or State    |
| Business Rule | Backend automation logic that runs automatically |
| Work Notes    | Internal support notes not visible to customers  |

## Things that confused me initially 😭

* Some fields like **Impact** and **Priority** were hidden because of the selected form view
* ServiceNow forms change based on roles/views
* After submitting an incident, finding it again from the list took me a minute because of filters/views

## What surprised me 🌌

ServiceNow isn’t just a ticketing tool.

It actually feels like a full ecosystem where:

* data,
* workflows,

are all connected together.

Kinda feels like the operating system of enterprise companies.
